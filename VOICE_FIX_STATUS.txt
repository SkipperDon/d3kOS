WHAT WAS FIXED
==================================================

ROOT CAUSE IDENTIFIED:
- USB audio card numbers change on Raspberry Pi reboot
- Voice assistant hardcoded to plughw:3,0 (card 3)
- After reboot, Anker S330 becomes card 2/4/5 instead
- System records from WRONG device → weak signal → Vosk fails

SOLUTION IMPLEMENTED:
✅ Modified voice-assistant-hybrid.py to use card NAME
✅ Now uses: plughw:S330,0 (persistent across reboots)
✅ Added re-detection before each listen
✅ Service running with new code
✅ Backup created: voice-assistant-hybrid.py.bak.usb-renumber

BONUS: Software gain boost attempted (.asoundrc)
