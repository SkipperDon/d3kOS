### 4. Fullscreen Toggle Fix (Wayland)

**Status:** âœ… COMPLETE

#### Problem Statement

After system reboot, the fullscreen toggle stopped working. The toggle is used when opening Engine Setup and Helm to exit fullscreen mode so the on-screen keyboard is accessible.

#### Root Cause Analysis

The system is running **Wayland** (labwc compositor) instead of X11:
- Window Manager: `labwc` (Wayland compositor)
- Display Server: Xwayland on DISPLAY :0
- Original script used `xdotool` which only works on X11
- `xdotool` cannot interact with Wayland windows properly

**Evidence:**
```bash
$ ps aux | grep labwc
d3kos  1373  1.5  1.3  504860  104304  ?  Ssl  14:22  0:55  /usr/bin/labwc -m

$ ps aux | grep Xwayland
d3kos  1697  0.0  1.0  303388  81516  ?  Sl  14:22  0:00  /usr/bin/Xwayland -auth /home/d3kos/.Xauthority :0
```

#### Solution: Install and Use wtype

**wtype** is the Wayland equivalent of xdotool for keyboard input.

**Installation:**
```bash
sudo apt-get install -y wtype
```

**Updated Script:** `/usr/local/bin/toggle-fullscreen.sh`
```bash
#!/bin/bash
# Toggle fullscreen for Chromium browser on Wayland
export XDG_RUNTIME_DIR=/run/user/1000
export WAYLAND_DISPLAY=wayland-0
wtype -k F11
```

**Why this works:**
- `wtype` sends keyboard events directly to Wayland compositor
- No need to find/focus specific windows
- Requires `XDG_RUNTIME_DIR` and `WAYLAND_DISPLAY` environment variables
- F11 key toggles fullscreen in Chromium

#### Lessons Learned

1. **Wayland vs X11:** Always check display server type before using X11 tools
2. **Tool Compatibility:** xdotool (X11), wtype (Wayland), ydotool (Wayland alternative)
3. **Environment Variables:** Wayland tools need `XDG_RUNTIME_DIR` and `WAYLAND_DISPLAY`
4. **Raspberry Pi Default:** Debian on Pi uses Wayland by default (labwc compositor)
